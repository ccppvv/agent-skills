# SOP 设计框架

## 概述

本技能提供系统化的SOP（标准化作业程序）设计方法，专门针对AI执行优化。核心框架：**AI-Ready SOP = 输入定义 + 处理流程 + 输出标准**，三大要素缺一不可。

## 何时使用

**触发场景：**
- 为AI助手设计可执行的工作流程
- 创建新人培训所需的标准化操作指南
- 将复杂任务拆解为可自动化的步骤
- 需要可迭代优化的业务流程
- 确保多人协作时结果一致性

**关键词：** SOP设计、流程标准化、AI工作流、操作手册、可复制流程

## 核心价值

设计出的SOP必须体现三大关键价值：

1. **可复制**：新人按SOP也能输出标准甚至专业的结果
2. **可优化**：每次执行后都可以收集反馈进行迭代改进  
3. **可自动化**：SOP本身就是AI的"操作手册"，可直接用于AI执行

## AI-Ready SOP 设计框架

### 要素一：输入定义 - 告诉AI需要什么信息

**输入内容：**
- 必须信息：完成任务必需的数据、参数、上下文
- 可选信息：有助于提高质量的补充信息
- 示例信息：用于模式识别的参考样例

**输入格式：**
- 结构化数据：JSON/YAML/表格格式
- 自然语言：特定领域的术语和表达
- 文件类型：支持的文档、图片、数据格式
- 验证规则：格式检查、完整性验证

**输入示例：**
```yaml
required_inputs:
  - name: "客户需求描述"
    format: "自然语言文本"
    validation: "至少包含产品类型和核心功能"
  
  - name: "预算范围"  
    format: "数值区间（如：1000-5000）"
    validation: "必须是合理价格区间"
    
optional_inputs:
  - name: "参考案例"
    format: "URL或文档链接"
    purpose: "风格参考和模式识别"
```

### 要素二：处理流程 - 告诉AI按什么步骤思考

**1. 自动化检测阶段：**
- 输入完整性检查
- 格式合规性验证
- 异常情况识别
- 风险初步评估

**2. 深度分析阶段：**
- 信息结构化处理
- 关键要素提取
- 模式识别与分类
- 关联性分析

**3. 风险评估阶段：**
- 实施难度评估
- 潜在问题预测
- 资源需求估算
- 替代方案分析

**结构化流程示例：**
```
1. 输入接收与验证
   ├── 检查必填字段
   ├── 验证数据格式
   └── 识别异常输入

2. 信息深度处理
   ├── 提取核心参数
   ├── 构建任务框架
   └── 确定优先级

3. 方案生成与评估
   ├── 生成可行方案
   ├── 风险评估
   └── 推荐最优方案
```

### 要素三：输出标准 - 告诉AI什么样的结果才算合格

**输出格式（必须包含四部分）：**

1. **总结**：整体执行结果概述
2. **问题清单**：发现的问题和风险点
3. **修改建议**：具体的改进建议和优化方向
4. **学习要点**：从本次执行中学到的新知识

**质量标准（按领域定义）：**

**技术开发领域：**
- 代码规范符合度 ≥95%
- 单元测试覆盖率 ≥85%
- 性能指标达标率 100%
- 安全漏洞数 = 0

**内容创作领域：**
- 格式规范评分 ≥90分
- 核心信息完整度 100%
- 可读性评分 ≥85分
- 原创性要求 ≥80%

**数据分析领域：**
- 数据准确率 ≥99.5%
- 分析深度符合预期
- 可视化清晰易懂
- 洞察价值评估 ≥8/10

## 设计步骤

### 第一步：领域分析
1. 确定SOP适用领域
2. 分析典型任务场景
3. 识别成功关键因素
4. 定义质量评价标准

### 第二步：输入规格设计
1. 列出所需输入信息
2. 定义格式和验证规则
3. 设计输入示例模板
4. 制定异常处理机制

### 第三步：处理流程构建
1. 分解任务为原子步骤
2. 定义每个步骤的逻辑
3. 建立步骤间依赖关系
4. 设置检查点和决策点

### 第四步：输出标准制定
1. 定义输出格式结构
2. 制定质量评估指标
3. 创建结果验证方法
4. 设计迭代改进机制

## 示例：技术代码审查SOP

**输入定义：**
```json
{
  "required": {
    "code_url": "GitHub/GitLab代码链接",
    "review_scope": "重点审查模块列表",
    "quality_standard": "团队代码规范文档"
  },
  "optional": {
    "previous_reviews": "历史审查记录",
    "team_guidelines": "团队开发指南"
  }
}
```

**处理流程：**
1. **自动化检测**：语法检查、依赖分析、复杂度计算
2. **深度分析**：架构评估、设计模式识别、性能分析
3. **风险评估**：安全漏洞扫描、可维护性评估、技术债识别

**输出标准：**
- **总结**：整体代码质量和改进建议
- **问题清单**：具体问题位置和描述
- **修改建议**：明确的重构方案
- **学习要点**：团队可复用的最佳实践

## 质量控制

使用SOP时，必须回答以下问题：

1. **可复制性检查**：新人能否仅凭SOP完成任务？
2. **可优化性检查**：SOP是否有明确的反馈收集点？
3. **可自动化检查**：AI能否直接执行每个步骤？

## 迭代改进

每次使用SOP后更新：
- ✅ 记录执行过程中的难点
- ✅ 收集结果的质量反馈  
- ✅ 识别可优化的步骤
- ✅ 更新模板和检查清单
