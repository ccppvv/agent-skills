## 产品经理Prompt

### 📝 PRD文档生成

```
你是一位资深产品经理。请基于以下需求信息，生成一份结构化的PRD文档。

## 需求背景
[描述业务背景和用户痛点]

## 目标用户
[描述目标用户画像]

## 核心需求
[列出核心功能需求]

## 约束条件
- 技术栈：[...]
- 时间要求：[...]
- 资源限制：[...]

请输出包含以下部分的PRD：
1. 需求概述（1-2段）
2. 用户故事（User Story格式）
3. 功能清单（优先级排序）
4. 交互流程（关键路径描述）
5. 非功能性需求（性能、安全、兼容性）
6. 验收标准（DONE定义）
7. 风险点与依赖
```

### 🎤 用户访谈整理

```
请分析以下用户访谈记录，提炼关键信息：

## 访谈对象
- 姓名/代号：[...]
- 用户画像：[职业/年龄/使用场景]

## 访谈记录
[粘贴转录文本或笔记]

请输出：
1. **核心需求**（用户真正想解决的问题，用"Jobs to be Done"框架描述）
2. **痛点场景**（具体在什么情况下遇到问题，包含频次和影响程度）
3. **期望解决方案**（用户明确提到的期望功能）
4. **潜在需求**（用户未明确提及但访谈中透露的隐性需求）
5. **引用原文**（支持以上分析的关键原话）

格式要求：每项以1-2句话概括，附上置信度（高/中/低）
```

### 🔍 竞品分析

```
请对以下竞品进行系统性分析：

## 竞品信息
- 产品名称：[...]
- 目标用户：[...]
- 核心功能：[...]

## 分析维度
请从以下维度进行分析：
1. **产品定位**：解决什么问题？目标用户是谁？
2. **核心功能**：有哪些关键功能？如何实现的？
3. **用户体验**：交互流程是否流畅？有哪些亮点设计？
4. **商业模式**：如何盈利？付费转化路径是什么？
5. **优势与不足**：对比我们产品的差异化点

最后输出：
- **SWOT分析**（Strengths, Weaknesses, Opportunities, Threats）
- **可借鉴点**（3-5个具体建议）
- **差异化策略**（我们可以如何做得不同且更好？）
```

## 运营Prompt

```
请帮我创作一篇公众号文章：

## 主题

## 目标受众
[读者画像：职业/年龄/关注点]

- [ ] 提升品牌认知
- [ ] 引导产品试用
- [ ] 建立专业形象

## 风格要求
- 语气：[专业/轻松/幽默/...]

- 结构：[总分总 / 故事化 / 列表式]

## 素材

1. **标题**（3-5个备选，包含悬念/数字/痛点元素）
```

## 开发工程师Prompt

### 🐛 Bug调试辅助

```
我遇到了以下错误，请帮我分析原因并提供修复建议：

## 错误信息
[粘贴完整错误堆栈]

## 相关代码
[粘贴触发错误的代码片段]

## 运行环境
- 语言/框架：[如 Python 3.11 / Django 5.0]
- 操作系统：[macOS / Linux / Windows]
- 相关依赖版本：[...]

## 复现步骤
1. [...]
2. [...]

## 已尝试的解决方法
- [方法1] → [结果]
- [方法2] → [结果]

请提供：
1. **根本原因分析**（为什么会出现这个错误）
2. **修复方案**（具体代码修改，标注修改位置）
3. **预防措施**（如何避免类似问题，如单元测试/code review要点）
4. **相关文档**（官方文档链接或最佳实践参考）
```

### 👀 Code Review

```
请对以下代码进行Code Review：

## 代码片段
[粘贴代码]

## 代码上下文
- 功能：[这段代码的作用]
- 语言/框架：[...]
- 代码库规范：[如有团队规范文档，请附上]

请从以下维度进行审查：
1. **潜在Bug**（逻辑错误、边界条件、并发问题）
2. **性能问题**（时间/空间复杂度、N+1查询、内存泄漏）
3. **代码规范**（命名、格式、注释）
4. **安全漏洞**（SQL注入、XSS、敏感信息泄露）
5. **可维护性**（是否过度复杂、重复代码、魔法数字）
6. **测试覆盖**（需要补充的测试用例）

输出格式：
- 严重性：🔴 Critical / 🟡 Warning / 🟢 Suggestion
- 行号：[...]
- 问题描述：[...]
- 修复建议：[具体代码]
```

## 通用工作流Prompt链

### Prompt链：完整内容创作工作流

**Step 1: 选题分析**
```
我需要创作[内容类型]，目标受众是[...]。
请基于以下信息推荐3-5个选题方向：
- 行业趋势：[...]
- 用户痛点：[...]
- 竞品内容gap：[...]

每个选题请提供：标题方向 + 核心观点 + 预期价值
```

**Step 2: 大纲生成**
```
选定选题：[...]
请生成详细内容大纲，包含：
1. Hook开篇（前100字吸引读者）
2. 3-5个核心要点（每个要点的论证逻辑）
3. 案例/数据支撑（每个要点至少1个）
4. CTA结尾（引导读者行动）
```

**Step 3: 初稿撰写**
```
基于以上大纲，撰写完整初稿。
要求：
- 语气：[...]
- 字数：[...]
- 穿插金句和小标题，便于阅读
```

**Step 4: 优化润色**
```
请优化以下初稿：
[粘贴初稿]

优化方向：
1. 标题更吸引人（A/B测试3个版本）
2. 开篇更有冲击力
3. 论点增加数据支撑
4. 删除冗余表达
5. 提炼2-3句金句用于传播
```

**1. 长文档分析**
- 使用Claude的200K上下文窗口优势
- Prompt示例：
```
请分析以下[合同/PRD/研究报告]，提炼：
1. 核心观点（3-5点）
2. 关键数据（用表格整理）
3. 风险点（需要关注的条款）
4. 行动项（下一步需要做什么）

[粘贴完整文档]
```

**2. 多步推理**
- 利用Claude的思维链能力
- Prompt示例：
```
请一步步分析：
1. 首先，识别问题的根本原因
2. 然后，评估3种可能的解决方案
3. 接着，对比方案的优劣势
4. 最后，给出推荐方案和实施路径

问题描述：[...]
```

### GPT-4最佳实践

**1. 图像理解（GPT-4V）**
```
[上传截图]
请分析这个界面设计：
1. 信息架构是否合理
2. 交互流程是否流畅
3. 视觉层级是否清晰
4. 给出3-5个具体优化建议
```

**2. 代码解释器（Advanced Data Analysis）**
```
[上传CSV数据]
请进行探索性数据分析：
1. 数据概览（行列数、数据类型、缺失值）
2. 异常值检测
3. 相关性分析（生成热力图）
4. 关键指标趋势（生成折线图）
5. 业务洞察和建议
```

### Midjourney最佳实践

**通用Prompt结构**
```
[主体描述] + [风格] + [视角/构图] + [光影] + [参数]

示例：
A modern tech startup office, minimalist design, wide angle view, soft natural lighting, bright and airy atmosphere --ar 16:9 --v 6
```

**岗位应用：**
- 运营：活动海报、社交媒体配图
- 设计师：概念设计、mood board
