### 漏斗转化率计算

```
咨询转化率 = 咨询人数 ÷ 引流人数 × 100%
付费转化率 = 付费人数 ÷ 咨询人数 × 100%
复购率 = 复购人数 ÷ 总付费人数 × 100%
```

## 常见迭代方案库

### 问题：引流不够

**症状**：每周加微信不到10人

**方案A：优化内容选题**
- 检查最近5条内容的互动数据
- 互动最高的那条选题方向，再写3条同类内容
- 选题公式：痛点+人群+方法

**方案B：换平台/加平台**
- 当前平台效果差 → 换一个目标用户更聚集的平台
- 当前平台效果还行 → 加一个平台做分发
- 每个新平台给2周测试期，没效果就砍掉

**方案C：付费测试**
- 用最小预算（100-300元）做一次精准投放
- 测试不同素材的效果
- ROI为正就加投，为负就停

### 问题：咨询不付费

**症状**：有人问但没人买

**方案A：增加信任背书**
- 收集3个客户案例/见证
- 在产品介绍中加入"他们买了之后怎样"
- 提供1次免费体验/诊断降低决策门槛

**方案B：优化成交话术**
- 不要推销，做需求诊断
- 话术流程：了解情况→诊断问题→推荐方案→消除顾虑→引导行动
- 给明确的下一步（"要不要试试？"而不是"你觉得呢？"）

**方案C：调价测试**
- 如果定价>500元 → 推出体验价（正常价50%）
- 如果定价<100元 → 反而考虑涨价（低价可能让人怀疑质量）
- 测试期1-2周，看转化率变化

### 问题：复购差

**症状**：客户买完就走

**方案A：7天回访机制**
- 交付后第3天：问使用情况
- 交付后第7天：问效果+收集反馈
- 交付后第14天：推荐下一个相关产品

**方案B：设计升级路径**
```
入门产品（低价）→ 进阶产品（中价）→ 深度服务（高价）
每个产品解决不同深度的问题
在入门产品交付时自然引出进阶产品
```

**方案C：建社群做留存**
- 付费客户专属社群
- 定期分享独家内容
- 社群本身就是复购的理由

### 问题：效率太低

**症状**：每天工作10小时但产出不多

**方案A：精力管理优先于时间管理**
- 保护每天最清醒的2小时，只做高价值决策
- 其余时间机械化执行
- 上午做创造型工作，下午做执行型工作

**方案B：砍掉低价值任务**
- 列出上周做的所有事
- 标记每件事的收入贡献（直接/间接/无）
- 砍掉"无"贡献的事，外包"间接"贡献的事

**方案C：引入AI工具**
- 参考 ai-workflow 模块的工具选型和数字员工配置
- 优先自动化最耗时的重复性工作
- 一次只引入1个工具，用顺了再加

## 复盘频率建议

| 业务阶段 | 复盘频率 | 重点看什么 |
|----------|----------|-----------|
| 冷启动期（0-3个月） | 每周 | 内容互动率、咨询量 |
| 增长期（3-6个月） | 每两周 | 付费转化率、客单价 |
| 稳定期（6个月+） | 每月 | 复购率、收入趋势 |

## 72小时情绪记录卡

当情绪低点想做大决定时，用这个卡片记录：

```
日期：____
我现在想做的决定：____________________
我的情绪状态（1-10分）：____
触发事件：____________________

→ 锁定72小时，不执行任何决定

72小时后复查：
日期：____
我的情绪状态（1-10分）：____
我还要做这个决定吗？ 是/否
如果"是"，经过冷静思考后的理由：____________________
```
