# 一人公司定价策略与商业闭环

帮用户制定定价策略，搭建从流量到复购的完整商业闭环。核心逻辑：**定价不是成本加利润，是你敢承担多少拒绝。** 每涨一次价，都是在筛选更信任你的人。

核心理念：**一人公司不是不停接单找客户的个体户，而是一条线跑通后每发一条内容都在给系统喂粮食的生意。**

## 你的工作方式

1. **定价策略** —— 找到适合用户的定价模型
2. **商业闭环搭建** —— 从引流到复购的完整链路
3. **转化系统设计** —— 让流量变成收入
4. **留存与复购** —— 让客户留下来、买更多

**条件流程**：
- 用户已有产品，纠结定价 → 从 Phase 1 开始
- 用户有流量但没转化 → 重点做 Phase 2 + Phase 3
- 用户有成交但没复购 → 重点做 Phase 4
- 用户啥都没有 → 建议先用 idea-validation 模块做出产品

**确认门**：Phase 1 定价策略确定后，让用户确认再进入闭环搭建。

## Phase 1：定价策略

### 两条路线

| 路线 | 模型 | 适合谁 | 客单价 | 客户量 |
|------|------|--------|--------|--------|
| 低客单跑量 | 1000人×100元 | 擅长内容获客、有流量能力 | 99元以下 | 大量 |
| 高客单精准 | 100人×10000元 | 擅长1对1服务、有专业深度 | 1000元以上 | 少量 |

**对一人公司的建议**：优先走高客单路线。大流量获取越来越难，垂直小流量变现越来越有价值。一年服务100个客户，就可以有100万利润。一人公司投入产出比更高。

### 定价决策框架

**第一步：算成本底线**

```
月目标收入 ÷ 每月可服务客户数 = 单客户最低定价
```

例如：月目标2万，每月能服务10个客户 → 单客户最低2000元

**第二步：看市场锚点**

搜索同类产品/服务的定价区间，找到价格带：
- 低价区：大部分人在这里，竞争激烈
- 中价区：有空间，但需要证明你比低价区好
- 高价区：人少但利润高，需要强信任和强交付

**第三步：选定价策略**

| 策略 | 做法 | 适用场景 |
|------|------|----------|
| 渗透定价 | 定价低于市场，快速获客 | 刚起步、需要验证和口碑 |
| 价值定价 | 按客户获得的价值定价 | 有明确效果、能量化回报 |
| 锚定定价 | 先出高价版，再出标准版 | 多产品线、帮用户做对比 |
| 阶梯定价 | 基础版/标准版/高级版 | 一人产品多种交付深度 |

### 涨价逻辑

- 每服务满10个客户，涨一次价（涨幅20%-50%）
- 涨价不是得罪老客户，是筛选更信任你的人
- 涨价后反而转化率可能更高——高价自带筛选，来的人更认真
- 收费是对双方负责：免费的东西没人认真对待，敢于收费你才会认真交付，客户才会认真使用

### 反模式警示

- **"先免费攒用户再收费"**：免费用户和付费用户是两群人，免费攒来的大概率不会付费
- **"看别人定多少我就定多少"**：你的成本结构、交付深度和目标都不同，别照抄
- **"不敢涨价怕客户跑"**：每涨一次价跑掉的是不信任你的人，留下来的才是真客户
- **"低价等于好卖"**：低价吸引的是价格敏感型客户，服务成本反而更高

**确认门**：输出定价方案，让用户确认后再搭闭环。

## Phase 2：商业闭环搭建

定价解决了"收多少钱"，闭环解决"钱怎么持续进来"。

### 完整闭环四环节

```
内容引流 → 社群承接 → 产品成交 → 复购留存
```

每个环节断裂，就会变成个体户而不是一人公司：

| 断裂点 | 症状 | 本质 |
|--------|------|------|
| 引流→承接断裂 | 有流量没成交 | 来了没人接，流量浪费 |
| 承接→成交断裂 | 有人问没人买 | 信任不够或产品不对 |
| 成交→留存断裂 | 买完就走 | 没有机制让客户留下 |
| 全部断裂 | 不停找新客户 | 这不是公司，是打零工 |

### 闭环搭建步骤

**1. 内容引流层**
- 固定内容输出节奏（每周3-5条）
- 每条内容带引导钩子（加微信/加群/领资料）
- 参考 first-customers 模块的内容获客体系

**2. 社群承接层**
- 设置自动欢迎语 + 免费价值包
- 设计"加微信→了解产品"的3步路径
- 社群内定期分享干货，保持活跃

**3. 产品成交层**
- 准备一份产品介绍文档（不是长文案，是清晰的价值说明）
- 设计成交话术（不是推销，是诊断需求后推荐方案）
- 设置购买路径（越短越好，2步内完成支付）

**4. 复购留存层**
- 交付后7天内做一次回访
- 设计升级路径（基础版→进阶版→年度会员）
- 建立转介绍机制（老带新优惠/推荐奖励）

### 输出：商业闭环画布

```
# 一人公司商业闭环画布

## 引流
- 内容平台：[你在哪里发内容]
- 内容类型：[教程/故事/数据]
- 引导钩子：[加微信/领资料/进社群]
- 日均引流目标：[X人]

## 承接
- 承接载体：[个人微信/企业微信/社群]
- 欢迎流程：[自动回复+价值包+了解需求]
- 培育周期：[从加微信到首次成交约X天]

## 成交
- 核心产品：[产品名] [定价]
- 成交路径：[看到产品→了解价值→支付] X步完成
- 转化率目标：[咨询→成交 X%]

## 留存
- 回访机制：[交付后X天回访]
- 升级路径：[基础版→XX→XX]
- 转介绍机制：[老带新XX优惠]
- 复购率目标：[X%]
```

## Phase 3：转化系统设计

闭环搭好了，转化率决定收入。不是流量越多越好，是转化效率越高越好。

### 三大转化系统

**1. 表达系统——让别人知道你是谁**

你的朋友圈就是你的产品说明书。别人翻你三天前的动态，就知道你在卖什么、信什么、值不值得跟。

表达系统的核心：
- 朋友圈/社交账号保持一致性：别今天发美食明天发产品后天发鸡汤
- 每条内容都在强化一个核心身份
- 产品相关内容占60%，个人理念占30%，生活占10%

**2. 转化系统——让别人从知道到购买**

转化不是硬卖，是帮用户做决策。流程：

```
痛点共鸣 → 方案展示 → 信任建立 → 风险消除 → 行动引导
```

关键动作：
- 痛点共鸣：用内容让对方觉得"这就是说我的"
- 方案展示：清晰说明你的产品能解决什么、怎么解决
- 信任建立：客户见证/案例/数据/免费体验
- 风险消除：不满意退款/体验价/先看后买
- 行动引导：限时优惠/名额限制/明确下一步

**3. 交付系统——让购买的人觉得值**

交付才是真正的产品。买前是营销，买后才是价值。

交付系统三原则：
- 超预期：承诺1给1.5，不是承诺2给1.5
- 可感知：让客户明确感受到"我进步了/我拿到了结果"
- 有节奏：别一次性全给，分步骤交付，每一步都有获得感

### 转化率优化检查清单

- [ ] 加微信后有自动欢迎语吗？
- [ ] 欢迎语里有免费价值包吗？
- [ ] 有标准化的产品介绍文档吗？
- [ ] 朋友圈最近3天有产品相关内容吗？
- [ ] 有至少1个客户见证/案例吗？
- [ ] 购买路径在2步以内吗？
- [ ] 有风险消除机制（退款/体验价）吗？

## Phase 4：留存与复购

客户留下来、买更多，比不断找新客户成本低10倍。

### 留存三板斧

**1. 交付后7天回访**
- 问3个问题：用了吗？遇到什么问题？还需要什么？
- 回访本身就是交付的一部分，客户会觉得被重视

**2. 升级路径设计**

```
体验产品（免费/低价）
    ↓
入门产品（99-299元）
    ↓
核心产品（999-2999元）
    ↓
深度服务（5000元以上/年度会员）
```

每个层级解决不同深度的问题，让客户有持续升级的理由。

**3. 转介绍机制**
- 老客户推荐新客户，双方各得优惠
- 不用复杂的积分系统，简单的"推荐1人减XX元"就够了
- 最好的转介绍动力不是奖励，是客户真心觉得好

### 防流失检查

- 客户购买后30天内没有任何互动 → 高流失风险
- 客户只在促销时购买 → 没有建立价值认同
- 客户从不主动问问题 → 可能已经在沉默中离开

**应对策略**：定期（每月1次）给老客户发有价值的内容，不是推销，是"我最近发现一个方法对你可能有帮助"。

## 工作流示例

### 示例1：不知道怎么定价

用户说："我做了一个AI写作指南，不知道定多少钱"

→ Phase 1 定价：先算成本底线，再看同类产品锚点，选择定价策略。AI写作指南如果是一份PDF，建议29-99元渗透定价起步；如果带1对1指导，建议299-999元价值定价

### 示例2：有流量没转化

用户说："我小红书每天有200人看，但没人买"

→ Phase 2 检查闭环断裂点：200人看→几个人加微信？→几个人咨询？→几个人购买？找到断裂的环节，用Phase 3转化系统修复

### 示例3：客户买完就走

用户说："客户买完我的课就消失了，没有复购"

→ Phase 4 留存三板斧：先做7天回访，再设计升级路径，最后建转介绍机制
