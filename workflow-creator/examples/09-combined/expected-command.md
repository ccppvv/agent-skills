# /multi-region-deploy - Multi-Region Deployment with Coordinated Rollback

> **Workflow Description**: Deploy to multiple regions in parallel with health verification, conditional production deployment, and coordinated global rollback on any failure

## Behavioral Flow

1. **Parallel Regional Deployment** (3 regions simultaneously)
   - Each region executes independently:
     - Deploy to staging environment
     - Wait for deployment stabilization
     - Run health verification checks

2. **Conditional Production Deployment** (per region)
   - If health check passes:
     - Deploy to production
     - Update shared status: "region_X: healthy"
   - If health check fails:
     - Rollback staging deployment
     - Update shared status: "region_X: unhealthy"

3. **Async Status Updates** (non-blocking)
   - Continuously update shared deployment status
   - Track progress across all regions
   - Provide real-time visibility

4. **Shared State Coordination**
   - Central deployment status tracker
   - Stores health status for all regions
   - Enables global decision making

5. **Global Health Evaluation**
   - Wait for all regional workflows to complete
   - Aggregate status from shared state
   - Make global deployment decision

6. **Coordinated Rollback** (if needed)
   - Trigger if any region reports unhealthy
   - Roll back all regions (even successful ones)
   - Ensure consistent state across infrastructure
   - Send failure notifications

## Boundaries

**Will:**
- Deploy to multiple regions in parallel for efficiency
- Verify health checks before production deployment in each region
- Coordinate global state across all regional deployments
- Automatically rollback all regions if any single region fails
- Maintain consistent deployment state across infrastructure
- Provide real-time status updates throughout process

**Will Not:**
- Deploy to production without health verification
- Allow partial deployments (all-or-nothing approach)
- Proceed if shared status coordination fails
- Skip rollback on failure (safety-first approach)
- Deploy without proper monitoring and alerting
