## 第一部分：输入定义质量检查

### 1.1 输入完整性检查
- [ ] **必填输入明确**: 所有完成任务必需的数据都已明确定义
- [ ] **可选输入有值**: 可选输入有明确的用途和价值说明
- [ ] **输入数量适当**: 既不过多（复杂性高），也不过少（信息不足）
- [ ] **没有冗余输入**: 每个输入都有不可替代的作用

### 1.2 输入格式检查
- [ ] **格式定义清晰**: 每个输入的格式都有明确说明
- [ ] **验证规则详细**: 输入验证的具体规则和方法
- [ ] **示例充分**: 提供正确和错误的输入示例
- [ ] **标准化程度**: 尽可能使用标准格式（JSON、YAML等）

### 1.3 输入质量保障
- [ ] **验证方法可行**: 验证规则可以实际执行
- [ ] **异常处理完备**: 对于各种异常情况都有处理方案
- [ ] **预处理步骤**: 有必要的输入预处理步骤
- [ ] **质量度量**: 定义了输入质量的评估指标

## 第二部分：处理流程质量检查

### 2.1 流程结构化检查
- [ ] **步骤原子化**: 每个步骤职责单一，可独立验证
- [ ] **步骤顺序合理**: 执行顺序有清晰的逻辑
- [ ] **依赖关系明确**: 步骤之间的依赖关系清晰
- [ ] **决策点清晰**: 所有决策点的条件和选项明确

### 2.2 逻辑完整性检查
- [ ] **无逻辑跳跃**: 没有从输入直接跳到结论的步骤
- [ ] **全覆盖路径**: 覆盖了所有可能的情况
- [ ] **异常流程**: 包含异常情况的处理流程
- [ ] **退出条件**: 清晰的流程终止或循环退出条件

### 2.3 质量控制点检查
- [ ] **质量门设置**: 在关键位置设置质量检查点
- [ ] **检查标准明确**: 每个质量门的检查标准具体可衡量
- [ ] **失败处理**: 质量检查失败后的处理方案
- [ ] **检查文档化**: 所有检查都留下记录

## 第三部分：输出标准质量检查

### 3.1 输出格式检查
- [ ] **四部分完整**: 包含总结、问题清单、修改建议、学习要点
- [ ] **格式标准**: 输出格式具有标准化结构
- [ ] **可读性强**: 输出内容易于阅读和理解
- [ ] **可操作性好**: 包含具体的行动建议

### 3.2 质量标准检查
- [ ] **标准可衡量**: 所有质量标准都可以量化评估
- [ ] **目标值合理**: 质量目标值既具有挑战性又是可实现的
- [ ] **评估方法可行**: 有切实可行的质量评估方法
- [ ] **优先级明确**: 不同质量维度的优先级明确

### 3.3 验证方法检查
- [ ] **自动检查覆盖**: 尽可能使用自动化检查工具
- [ ] **人工评审点**: 需要人工评审的关键点明确
- [ ] **验证流程完备**: 从验证到反馈的完整流程
- [ ] **改进机制**: 验证发现问题的改进机制

## 第四部分：可复制性检查

### 4.1 新人友好性
- [ ] **背景知识完整**: 提供完成任务所需的背景知识
- [ ] **术语解释清晰**: 所有专业术语都有解释
- [ ] **常见问题解答**: 包含常见问题和解决方案
- [ ] **学习路径**: 有明确的学习和上手路径

### 4.2 工具和模板
- [ ] **模板可用**: 提供标准化的输入输出模板
- [ ] **工具支持**: 提供必要的工具和脚本支持
- [ ] **示例充分**: 提供完整的实际执行示例
- [ ] **文档完整**: 所有工具和模板都有说明文档

### 4.3 错误处理支持
- [ ] **错误诊断指南**: 提供错误诊断和排查方法
- [ ] **恢复方案**: 出现错误后的恢复和重启方案
- [ ] **求助渠道**: 明确遇到困难时的求助渠道
- [ ] **经验分享**: 包含过往的经验教训总结

## 第五部分：可优化性检查

### 5.1 反馈收集机制
- [ ] **反馈点设置**: 在关键位置设置反馈收集点
- [ ] **反馈类型多样**: 收集不同类型的反馈（质量、效率、满意度等）
- [ ] **数据收集自动化**: 尽可能自动化收集执行数据
- [ ] **分析工具**: 提供数据分析工具和报告

### 5.2 改进流程
- [ ] **改进触发机制**: 明确什么情况下启动改进
- [ ] **根本原因分析**: 有系统性的问题分析流程
- [ ] **改进实施流程**: 从分析到实施的完整流程
- [ ] **效果评估**: 改进方案的效果评估方法

### 5.3 知识管理
- [ ] **学习记录**: 记录每次执行的学习要点
- [ ] **最佳实践更新**: 及时更新最佳实践案例
- [ ] **知识共享**: 有机制在团队内部分享知识
- [ ] **版本控制**: SOP和相关资源的版本管理

## 第六部分：可自动化检查

### 6.1 AI执行友好性
- [ ] **输入标准化**: 所有输入都有标准格式，易于AI解析
- [ ] **逻辑结构化**: 处理逻辑清晰结构化，无模糊判断
- [ ] **输出规范化**: 输出有固定格式，易于AI生成
- [ ] **异常明确化**: 所有异常都有明确处理逻辑

### 6.2 自动化障碍识别
- [ ] **人工判断识别**: 识别出需要人类经验判断的点
- [ ] **外部依赖**: 识别出不可控的外部系统依赖
- [ ] **模糊标准**: 识别出需要主观评价的质量标准
- [ ] **创新需求**: 识别出需要创新思维的环节

### 6.3 自动化路径规划
- [ ] **渐进自动化方案**: 从部分到全面的自动化路径
- [ ] **人工增强方案**: 人机协作的优化方案
- [ ] **技术可行性**: 现有技术条件下的自动化可行性
- [ ] **投入产出比**: 自动化的预期收益和成本
