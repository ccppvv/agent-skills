## 场景1: 绩效不达标员工

### 适用条件
- KPI 完成率 < 70%
- 重大项目失误
- 多次反馈无改进

### 完整脚本

#### 🎬 开场 (5分钟) - 严肃直接型

```
"[姓名],今天我们需要严肃地谈谈你的绩效问题。

这次是预反馈,正式评估在 [X周] 后。我需要坦诚告诉你,
目前你的表现没有达到岗位要求,如果接下来几周没有明显改进,
会影响到你的绩效评级、奖金,甚至可能影响到你的转正/续约。

但现在还有时间,我希望通过今天的沟通,明确问题所在,
制定改进计划,我也会提供必要的支持。

我们先从你的视角开始 - 你自己怎么评价这段时间的工作?"

【倾听 3-5 分钟,记录关键信息】
```

---

#### ⭐ 正向反馈 (5分钟) - 简短但真诚

```
"在说问题之前,我想先肯定你这些方面:

1. [找 1-2 个真实优点]
   例如:"你在技术细节上很认真,代码注释写得很详细"
   → 影响:"这帮助了新人快速理解代码逻辑"

2. [如果实在找不到亮点,可以说态度]
   "你对反馈的接受态度还是比较开放的,上次我提的建议你有在尝试改进"

但是,这些优点还不足以弥补核心问题。接下来我们重点谈谈
需要改进的地方。"
```

---

#### ⚠️ 改进反馈 (20分钟) - 核心重点

**【问题 1: 最严重的核心问题】**

```
"最需要解决的问题是 [核心问题名称]:

【S - Situation】
[具体项目/时间/场景]

【B - Behavior】
[客观描述做了什么/没做什么,用数据说话]

【I - Impact】
业务影响: [量化损失 - 延期/成本/客户投诉]
团队影响: [对同事的影响 - 返工/加班/士气]
个人影响: [对你信誉的影响 - 被质疑/失去机会]

【期望】
我期望你未来 [具体行为标准]:
- 标准 1: [可衡量的指标]
- 标准 2: [可衡量的指标]

你对这个问题怎么看? 有什么想补充的?"

【倾听,不打断,但坚持核心问题】
```

**【问题 2-3: 其他重要问题】**
(同样使用 SBI 模型,每个问题 5-7 分钟)

---

#### 💬 双向讨论 (10分钟)

```
"我们刚才谈了 [X个] 核心问题。我想听听:

1. 这些反馈,哪些你认同? 哪些你觉得需要澄清?
2. 在解决这些问题时,你觉得最大的困难是什么?
3. 你需要什么样的支持?"

【关键】
- 对合理的客观困难表示理解
- 但不接受推卸责任的借口
- 如果对方说"我尽力了":
  → "我理解你很努力,但结果仍然不达标。
     我们需要找到方法改变现状,而不只是投入时间。"
```

---

#### 📋 行动计划 (15分钟) - 具体到每周

```
"接下来 [X周] 是关键期,我们一起制定改进计划:

【问题 1: [核心问题]】

改进行动:
Week 1: [第1周要完成的具体任务]
Week 2: [第2周任务]
Week 3-4: [后续任务]

衡量标准:
- 最低要求: [底线标准]
- 理想目标: [期望标准]

检查频率:
- 每周一 1-on-1 (30分钟)
- 我会每天查看你的工作日报
- 第 2 周末进行中期复盘

我的支持:
- [具体支持 1: 培训/资源/时间]
- [具体支持 2]
- [具体支持 3]

【问题 2-3: 同样结构】

你觉得这个计划可行吗? 有什么调整建议?"

【一起在白板/文档上写下计划,双方签字确认】
```

---

#### 🔚 严肃收尾 (5分钟)

```
"[姓名],我需要再次强调:

1. 这些问题已经影响到了业务和团队,不是小问题。

2. 正式评估时,我会综合评估改进情况:
   - 如果有明显进步 → 评级可能调整为 [XX]
   - 如果改进不明显 → 评级会是 [XX],影响奖金/转正

3. 但我相信你有能力改变现状。你在 [某个优势] 上已经
   证明了自己,现在需要把这种能力应用到其他方面。

4. 接下来几周我会密切跟进,有任何问题随时找我。
   但最终,改变要靠你自己的行动。

你还有什么问题或担忧想说的吗?"

【最后握手/点头】
"好,那我们下周一 1-on-1 见,到时候我想看到你在
[第一个行动] 上的进展。"
```

---

#### 📧 会后 24h 邮件模板

```
Subject: 预反馈会议纪要 - [日期]

[姓名],

感谢今天的坦诚沟通。为确保我们对改进计划达成共识,
我整理了会议要点:

【核心问题】
1. [问题 1]: [一句话描述]
2. [问题 2]: [一句话描述]
3. [问题 3]: [一句话描述]

【改进计划】
见附件: "改进行动计划.xlsx"
(包含每周任务、衡量标准、检查时间点)

【我的承诺】
- 每周一 10:00-10:30 固定 1-on-1
- 提供 [具体支持资源]
- 第 2 周五进行中期复盘

【关键时间点】
- 第 1 周检查: [日期]
- 中期复盘: [日期]
- 正式评估: [日期]

【重要提醒】
正式评估时,改进情况将直接影响你的绩效评级和奖金。
我相信你能够改变现状,但需要你的实际行动。

有任何问题随时找我。

Best,
[你的名字]
```

## 场景2: 优秀员工发展指导

### 适用条件
- KPI 完成率 > 120%
- 有晋升潜力
- 主要目的是发展而非纠错

### 完整脚本

#### 🎬 开场 (5分钟) - 认可+展望型

```
"[姓名],首先我想说,这段时间你的表现非常出色,
超出了我对这个岗位的期望。

今天这次预反馈,重点不是纠正问题,而是和你聊聊:
1. 你超预期的亮点在哪里
2. 如果想进一步发展(比如晋升 [下一级别]),
   还需要在哪些方面提升
3. 我能提供什么支持

我们先从你自己的视角开始 - 你怎么评价这段时间的工作?
有没有特别有成就感的项目?"

【倾听 5-10 分钟,鼓励对方多说】
```

---

#### ⭐ 正向反馈 (15分钟) - 详细且真诚

```
"你在这些方面表现出色,我想具体说说:

【亮点 1: [具体成就]】
S: [项目/场景]
B: [你做了什么]
I: [对业务/团队的积极影响]
→ 这体现了你的 [核心能力标签: 如"系统思维"、"技术深度"等]

【亮点 2: [具体成就]】
[同上结构]

【亮点 3: [超预期表现]】
S: [场景]
B: [行为]
I: [影响]
→ 这个已经达到 [更高级别] 的要求了

特别要说的是 [某个项目],这个项目的 [某个方面] 让我印象深刻,
不仅解决了当下问题,还为团队建立了 [长期价值]。

你在 [核心能力] 上已经是团队 top 水平。"

【关键】给足细节,让对方感受到你真的关注了他的工作
```

---

#### 🚀 发展性反馈 (15分钟) - 不是纠错,是"更上一层楼"

```
"既然你表现这么好,我们来谈谈下一步发展。

如果你的目标是 [晋升到 Senior/Staff/etc.],
根据公司的晋升标准,有几个方面还需要提升:

【发展点 1: [能力维度]】
现状: [你目前的水平]
目标: [下一级别的要求]
差距: [具体差在哪里]

建议行动:
- [具体建议 1]
- [具体建议 2]
预期成果: [如果做到了,会有什么结果]

示例:

【发展点 1: 技术影响力】
现状: 你的技术方案在团队内部很受认可,但跨团队影响力还不够
目标: Staff 级别需要在公司/行业有一定技术影响力
差距: 目前你还没有跨部门的技术方案经验,也没有对外分享

建议行动:
- Q3 主导 1 个跨部门技术方案(我可以推荐你参与 XX 项目)
- Q3-Q4 在公司技术大会做 1 次分享
- Q4 在团队博客/公众号发表 2-3 篇技术文章
预期成果: 让公司更多人知道你的技术能力,为晋升答辩积累素材

【发展点 2: 人才培养】
现状: 你自己能力很强,但还没有带过新人
目标: Senior+ 需要有 mentor 经验,能培养团队梯队
差距: 目前你主要是独立完成任务,协作较少

建议行动:
- Q3 开始 mentor 1 个初级工程师 (我会分配 XX 给你带)
- 参与 2-3 次校招面试,学习识别人才
- Q4 在新人培训中做 1 次技术分享
预期成果: 展现 leadership 潜力,这是晋升的重要加分项

你对这些发展建议怎么看? 有没有你特别感兴趣或担心的?"

【倾听,鼓励对方提出自己的想法】
```

---

#### 💡 双向讨论 (10分钟)

```
"我刚才说的是我观察到的发展方向。我也想听听:

1. 你自己对未来 1-2 年的职业目标是什么?
   (晋升? 转岗? 深耕技术? 转管理?)

2. 有没有你觉得自己的短板或想提升的地方?

3. 有什么阻碍你发展的因素?(项目资源、时间分配等)

【根据对方回答,调整发展计划】
```

---

#### 📋 发展计划 (10分钟)

```
"好,基于我们的讨论,我们一起制定一个发展计划:

【目标】: 在 [X个月] 内,达到 [下一级别] 的晋升标准

【关键里程碑】
Q3:
- [里程碑 1]: [具体任务 + 衡量标准]
- [里程碑 2]

Q4:
- [里程碑 3]
- [里程碑 4]

【我的支持】
- 项目机会: 我会让你主导 [XX 项目],这是展示能力的好机会
- 曝光机会: 推荐你参与 [跨部门会议/分享会]
- 学习资源: [课程/书籍/导师推荐]
- 时间保障: 每月 20% 时间用于这些发展性任务,不算 KPI

【检查频率】
- 双周 1-on-1 时回顾进展
- 季度末做一次全面复盘

【晋升时间线】
- 如果进展顺利,我会在 [时间] 提名你参加晋升评审
- 晋升答辩的 PPT 我会帮你一起准备

你觉得这个计划如何? 压力大不大?"

【一起调整计划,确保对方觉得挑战但可达成】
```

---

#### 🎉 鼓励收尾 (5分钟)

```
"[姓名],我对你的未来发展非常看好。你已经证明了自己
在 [核心能力] 上的实力,接下来只是把这种能力扩展到
更大的范围。

几点建议:
1. 不要给自己太大压力,发展是长期的,不是短跑
2. 遇到困难随时找我,我见过很多类似的情况
3. 保持现在的学习热情和开放心态,这是你最大的优势

正式评估时,我会给你 [优秀/超预期] 的评级。但更重要的是,
我希望你能持续成长,未来成为团队的 [技术 leader/骨干]。

你还有什么想聊的吗?"

【握手/击掌】
"太好了,那我们下次 1-on-1 继续聊发展计划的细节。
期待看到你更多的成长!"
```

## 场景3: 防御型员工

### 识别特征
- 听到批评立刻找借口
- 归因外部("不是我的问题")
- 情绪激动,打断解释

### 特殊技巧

#### 📌 核心原则
1. **证据要充足** - 用数据和事实,不给辩驳空间
2. **预设缓冲问句** - 主动给对方解释机会
3. **坚持核心问题** - 理解客观困难,但不接受推卸责任

#### 🎬 开场调整

```
"[姓名],今天我们聊聊你的工作表现。

在开始之前,我想说:
1. 我准备了具体的数据和案例,不是主观评价
2. 我也想听听你的视角,可能有我不了解的情况
3. 我们的目标是一起找到改进方法,不是指责

【关键】先打预防针,降低防御心理
```

---

#### ⚠️ 改进反馈 - 增强版

**每个问题后加"缓冲问句"**:

```
【标准 SBI】
S: [情境]
B: [行为]
I: [影响]

【缓冲问句】👈 关键!
"我说的这些,你怎么看?
有没有我不了解的背景因素?
或者你觉得哪里不够准确?"

【倾听 3-5 分钟】
- 不打断
- 记录对方的解释
- 表情保持中立

【回应策略】
如果解释合理:
"好的,这个情况我确实不知道。考虑到这个因素,
 我们一起想想怎么在这种约束下做得更好?"

如果解释是推卸责任:
"我理解 [客观困难],这确实是一个因素。
 同时,我们还是需要讨论:在这种情况下,
 你能做些什么来改善结果?"

【坚持核心问题】
"即使考虑到 [对方说的困难],结果仍然是 [负面影响]。
 我们需要找到方法避免类似情况再次发生。"
```

---

#### 💬 应对典型防御话术

| 防御话术 | ❌ 不好的回应 | ✅ 推荐回应 |
|---------|--------------|-----------|
| "这不是我的问题,是XX的锅" | "那你为什么不..." | "我理解有外部因素。同时,作为负责人,你有责任协调解决。我们来看看你能控制的部分。" |
| "我已经尽力了!" | "尽力不够,要结果" | "我相信你投入了时间。但'尽力'和'有效'是两回事。我们需要找到更有效的方法。" |
| "你不了解实际情况!" | "我当然了解..." | "你说得对,我可能有些细节不清楚。请详细说说,我想听听你的视角。" (倾听后仍坚持核心问题) |
| "上次也不是我的问题" | "我们今天不谈上次" | "我记得上次的情况。今天我们聚焦这次,然后看看有没有共同pattern。" |
| **沉默不回应** | "你怎么不说话?" | "我知道这些话可能让你不舒服。你可以先想想,我们可以休息 2 分钟。" |
| **情绪激动** | "你冷静点!" | "我看到你情绪比较激动,我们可以暂停一下。这些反馈不是针对你个人,是针对工作结果。" |

## 场景4: 敏感型员工

### 识别特征
- 容易哭泣或情绪低落
- 过度自责("我太差了")
- 对批评反应强烈

### 特殊技巧

#### 📌 核心原则
1. **三明治反馈法** - 正向→改进→正向
2. **软化语言** - "我观察到"代替"你总是"
3. **频繁鼓励** - 强调"能改进"而非"不够好"

#### 🎬 开场调整 - 温和型

```
"[姓名],我们今天聊聊你的工作表现。

首先我想说,这次沟通的目的是帮助你成长,不是批评你。
我看到了你的很多优点,也看到了一些可以提升的空间。
我相信通过我们的沟通,你会变得更好。

在开始之前,我想问问你:最近工作压力大吗?
有没有什么困扰想和我聊聊?"

【关键】先关心对方状态,建立安全感
```

---

#### ⚠️ 改进反馈 - 柔化版

**使用"我观察到"语言模式**:

```
【柔化前】❌
"你的代码质量不行,总是有 bug"

【柔化后】✅
"我观察到你最近提交的 5 个 PR 中,有 3 个被 reviewer
指出了一些可以避免的 bug,比如空指针判断。"

【进一步柔化】✅✅
"我注意到你在代码细节上还有提升空间。比如最近几个 PR,
reviewer 提到了空指针判断的问题。这不是大问题,很多新人
都会遇到,我们一起想办法改进就好。"
```

**标准 SBI + 柔化语言**:

```
【S】[情境]

【B】[行为]
"我观察到..." / "我注意到..." / "我发现..."
(而非"你总是..." / "你从不...")

【I】[影响]
"这可能会导致..." / "这影响了..."
(而非"这造成了严重的..." / "你让大家...")

【鼓励】👈 额外增加!
"但我相信这是可以改进的,因为你在 [其他方面] 做得很好"
```

---

#### 💬 应对情绪反应

**如果对方开始哭泣**:

```
【第 1 步:暂停 + 递纸巾】
[递纸巾,沉默 10-20 秒]

【第 2 步:共情】
"我理解这些反馈让你不舒服。我不是要否定你,
 只是想帮助你成长。我们可以休息几分钟。"

【第 3 步:等待对方情绪稳定】
[沉默 1-2 分钟,让对方擦眼泪、平复]

【第 4 步:继续,但降低强度】
"我们继续聊吗? 还是改天再谈?"
(如果对方愿意继续)
"好的。我们放慢节奏,你随时可以说需要休息。"

【第 5 步:后续反馈减少 30%】
原计划说 5 个问题 → 只说 2-3 个核心问题
原本 20 分钟 → 缩短到 10 分钟
```

**如果对方过度自责**:

```
员工:"我太差了,什么都做不好..."

❌ 不要说:"也没有那么差..."(否认对方感受)
❌ 不要说:"你要对自己有信心"(空洞鼓励)

✅ 推荐说:
"我理解你现在有这种感觉。但客观来说,你在 [XX] 方面
 做得很好,这证明你是有能力的。现在只是需要在 [YY] 方面
 再提升一点。我见过很多人遇到类似问题,都改进了,
 你也可以做到。"

【策略】:
1. 承认对方感受
2. 用具体事实纠正过度自责
3. 举例证明"可以改变"
```

---

#### 🎉 收尾 - 强化鼓励

```
"[姓名],我们今天谈了一些需要改进的地方,
但我想再次强调:

1. 你有很多优点:[列举 2-3 个具体优点]
2. 你现在遇到的问题,很多人都经历过,这是成长的一部分
3. 我对你有信心,因为你 [具体证据: 比如"学习能力很强"]
4. 我会一直支持你,遇到困难随时找我

正式评估时,我会综合考虑你的进步。只要你在接下来几周
有改进,评级不会受太大影响。

最后问一句:你现在感觉怎么样? 压力会不会太大?"

【根据对方回答,再次安抚或调整计划】
```

## 场景5: 中等绩效员工

### 适用条件
- KPI 完成率 70-90%
- 没有严重问题,但也没有亮点
- 目标是"从合格到良好"

### 完整脚本

#### 🎬 开场 (5分钟) - 平衡型

```
"[姓名],我们今天聊聊你这段时间的工作表现。

整体来说,你的表现是符合预期的,完成了基本职责。
同时,我也看到一些可以进一步提升的空间 - 如果你想
在下次评估中拿到更好的评级,或者未来有晋升的目标。

今天的沟通主要是:
1. 肯定你做得好的地方
2. 探讨可以做得更好的地方
3. 一起制定一个提升计划

你自己怎么评价这段时间的工作? 有没有特别满意或不满意的?"

【倾听 5 分钟】
```

---

#### ⭐ 正向反馈 (10分钟) - 真诚且具体

```
"你在这些方面表现不错:

1. [具体优点 1]
   S: [情境]
   B: [行为]
   I: [影响]
   → 这体现了你的 [能力]

2. [具体优点 2]
   [同上]

3. [具体优点 3]
   [同上]

整体来说,你在 [核心职责] 上是可靠的,我交给你的任务
基本都能按时完成,这一点我很认可。"
```

---

#### 📈 改进反馈 (15分钟) - 成长导向

**框架:不是"纠错",是"从良好到优秀"**

```
"如果你想在下次评估中拿到更高评级,或者未来有晋升目标,
有几个方面可以进一步提升:

【提升点 1: [能力维度]】
现状: [你现在做到了什么程度]
差距: [和"良好"或"优秀"的差距]
影响: [如果提升了,会有什么好处]

建议:
- [具体建议 1]
- [具体建议 2]

示例:

【提升点 1: 主动性】
现状: 你交代的任务都能完成,但较少主动提出改进建议
差距: "良好"级别的员工会主动发现问题并提出解决方案
影响: 如果你能更主动,会承担更重要的项目,加速成长

建议:
- 每周在 1-on-1 时,主动提 1-2 个你发现的问题和建议
- 主动 review 其他同事的 PR,提改进建议
- 下个项目中,主动承担技术方案设计(我会指导你)

【提升点 2-3: 同样结构】

你对这些提升方向怎么看? 有没有你特别想提升的方面?"

【倾听,鼓励对方提出自己的目标】
```

```
  衡量标准: [如何判断完成]

【预期结果】
如果按计划执行,正式评估时你的评级有望从 [当前] 提升到 [目标]

```

```
接下来几周,我们一起努力,看看能把你的能力发挥到什么程度。
```

## 通用技巧总结

### ✅ 所有场景都适用的原则

1. **准备充分**: 提前收集数据,写下 SBI 案例
2. **私密空间**: 必须在会议室,关门,不被打扰
3. **时间控制**: 45-60 分钟为宜,不要超过 90 分钟
4. **记录**: 准备笔记本,记录关键信息和承诺
5. **会后跟进**: 24h 内发邮件总结,设置日历提醒

### ❌ 所有场景都要避免的错误

1. 🚫 在公开场合反馈
2. 🚫 没有准备,即兴发挥
3. 🚫 只谈问题不给方案
4. 🚫 情绪化表达("我对你很失望")
5. 🚫 翻旧账(超过 3 个月前的事)
6. 🚫 与其他同事对比("你看某某...")
7. 🚫 反馈后不跟进
