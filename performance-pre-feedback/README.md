## 📋 完整工作流

### Step 1: 场景识别 (2-3分钟)
技能会问你几个问题:

```
1. 员工当前绩效状态?
   □ 优秀/超预期
   □ 符合预期/中等
   □ 接近预期/需改进
   □ 不达标/问题严重

2. 主要反馈目的?
   □ 正向激励+发展指导
   □ 问题纠正+改进计划
   □ 期望管理+能力提升
   □ 绩效警示+最后机会

3. 员工性格特征?(多选)
   □ 防御型(容易抵触批评)
   □ 自驱型(主动寻求反馈)
   □ 敏感型(情绪容易波动)
   □ 成熟型(理性接受反馈)
   □ 逃避型(回避问题讨论)
```

**AI 会根据你的回答**:
- 自动匹配最合适的对话模板
- 评估沟通风险等级
- 推荐沟通策略

---

### Step 2: 数据收集指导 (5-10分钟)
技能会生成 `feedback-data-checklist.md`,包含:

**通用数据项**:
```
□ OKR/KPI 完成率
□ 关键项目产出
□ 正向贡献事件 (至少 2-3 个)
□ 问题事件记录 (具体时间+影响)
□ 360度反馈
□ 上次反馈后的改进情况
```

**场景化数据**(根据你的情况定制):
- 优秀员工 → 重点收集"成长瓶颈"和"更高目标"
- 问题员工 → 重点收集"业务影响"和"改进空间"量化数据
- 敏感型员工 → 准备"正面案例"平衡负面反馈

---

### Step 3: 对话脚本生成 (10-15分钟)
技能会生成完整的对话脚本 `feedback-script-[姓名].md`,包含:

#### 📝 脚本结构
1. **开场白** (5分钟) - 根据性格类型调整(温和/直接)
2. **正向反馈** (10分钟) - 使用 SBI 模型的具体案例
3. **改进反馈** (15-20分钟) - 每个问题都用 SBI 框架
4. **双向讨论** (10分钟) - 开放式提问,倾听技巧
5. **行动计划** (10分钟) - 细化到每周的改进任务
6. **积极收尾** (5分钟) - 鼓励型/严肃型(根据情况)

#### 🎯 特殊处理
- **防御型员工**: 每个问题后加"缓冲问句",给解释机会
- **敏感型员工**: 使用"三明治反馈法",柔化语言
- **问题严重员工**: 明确后果,严肃收尾

---

### Step 4: 心理风险评估 (5分钟)
技能会评估沟通风险,生成 `risk-assessment-[姓名].md`:

#### 风险维度 (0-10分)
```
防御反应风险: ██████░░░░ 6/10
情绪崩溃风险: ███░░░░░░░ 3/10
被动接受风险: ████░░░░░░ 4/10
```

#### 应对策略
```
【高防御风险 → 应对】
- 准备充足数据,不给辩驳空间
- 使用"缓冲问句":"你怎么看?"
- 坚持核心问题,不接受推卸责任

【高情绪风险 → 应对】
- 准备面巾纸,私密会议室
- 安抚话术:"我理解这些话让你不舒服..."
- 如果哭泣,暂停 2-3 分钟

【被动接受风险 → 应对】
- 行动计划具体到每日任务
- 每周检查,不等正式评估
- 频繁跟进,外部压力推动
```

---

### Step 5: 跟进计划生成 (5分钟)
技能会生成 `follow-up-plan-[姓名].md`,包含:

```markdown
## 即时行动 (会后 24h)
- [ ] 发送会议纪要邮件
- [ ] HR 系统记录
- [ ] 设置日历提醒

## 第 1 周检查点
时间: [日期]
检查内容:
- [ ] 是否开始执行改进行动?
- [ ] 有无预期外困难?
- [ ] 情绪状态是否稳定?

## 第 2-3 周检查点
[...]

## 第 4 周总结
- 对比初始问题,改进程度?
- 正式评估等级是否调整?
```

## 📂 输出文件说明

技能执行完毕后,会在项目目录生成:

```
performance-feedback-[日期]/
├── feedback-data-checklist.md       # 数据收集清单
├── feedback-script-[姓名].md        # 完整对话脚本 ⭐ 核心
├── risk-assessment-[姓名].md        # 心理风险评估
├── follow-up-plan-[姓名].md         # 跟进计划
└── quick-reference.md               # 快速参考卡片 ⭐ 打印使用
```

### 🌟 重点文件

#### 1. `feedback-script-[姓名].md`
- **用途**: 对话前通读 2-3 遍,熟悉话术
- **特点**: 包含完整的开场-反馈-行动计划-收尾
- **建议**: 打印出来,对话时放在桌上参考

#### 2. `quick-reference.md`
- **用途**: 对话时快速查看要点
- **特点**: 浓缩版,只有关键话术和数据
- **建议**: A4 纸打印,对话时放在手边

**示例**:
```markdown
# 快速参考 - 张工

## 3 个必说的优点
1. 技术攻坚能力强 - 订单系统优化(800ms→200ms)
2. 代码质量高 - PR 很少有 bug
3. 解决遗留问题 - 修复 2 个遗留 bug

## 3 个核心问题
1. Code Review 沟通方式不当 - "代码太烂了"
2. 跨部门协作否定式沟通 - "这不可能实现"
3. 团队协作满意度低 - 调研中 3 人提到不想合作

## 关键话术
如果抵触: "我理解你的出发点,但表达方式需要改进..."
如果沉默: "我想听听你的想法,你怎么看?"
```

## 💡 使用技巧

### ✅ 准备阶段 (对话前 1 周)

1. **收集数据** (使用 checklist)
   - 至少 2-3 个正面案例
   - 至少 2-3 个问题案例(用 SBI 格式记录)
   - 量化数据(KPI、客户投诉数、延期天数等)

2. **熟悉脚本** (至少 2 遍)
   - 第 1 遍:通读全文,理解结构
   - 第 2 遍:标注重点,记住关键话术
   - [可选] 第 3 遍:和同事演练(角色扮演)

3. **预演场景** (10-15分钟)
   - 想象对方可能的反应
   - 准备应对话术
   - 查看风险评估报告,准备应对策略

---

### ✅ 对话阶段 (45-60分钟)

1. **环境准备**
   - ✅ 私密会议室(不被打扰)
   - ✅ 准备纸巾(如果是敏感型员工)
   - ✅ 打印 quick-reference.md 放桌上
   - ✅ 准备笔记本,记录关键信息

2. **时间分配**
   - 开场: 5 分钟
   - 正向反馈: 10 分钟
   - 改进反馈: 15-20 分钟 (核心)
   - 双向讨论: 10 分钟
   - 行动计划: 10 分钟
   - 收尾: 5 分钟
   - **总计**: 55-60 分钟

3. **记录要点**
   - 对方的解释和理由
   - 对方承诺的改进行动
   - 你承诺提供的支持
   - 意外情况(情绪反应、新信息等)

---

### ✅ 会后阶段 (24h内)

1. **发送会议纪要邮件** (使用脚本中的模板)
   - 总结核心问题
   - 列出行动计划
   - 明确检查时间点
   - 附上改进计划表格

2. **HR 系统记录**
   - 预反馈已完成
   - 核心问题摘要
   - 下次检查时间

3. **设置日历提醒**
   - 第 1 周检查点
   - 第 2 周检查点
   - 第 4 周总结
   - 正式评估时间

---

### ✅ 跟进阶段 (持续 4 周+)

1. **每周检查** (使用 follow-up-plan.md)
   - 按计划检查改进进度
   - 记录进展和问题
   - 及时给予反馈(正向或纠偏)

2. **调整计划** (如需要)
   - 如果进展快 → 提高目标
   - 如果遇到困难 → 提供额外支持
   - 如果无进展 → 升级管理(HR介入)

3. **正式评估准备**
   - 回顾 4 周数据
   - 对比初始问题和当前状态
   - 决定最终评级
   - 准备正式评估对话

## ⚠️ 常见问题 FAQ

### Q1: 技能生成的时间太长?
**A**: 完整流程需要 10-15 分钟。如果只需要某个部分:
```
"我只需要对话脚本,不需要风险评估和跟进计划"
```

### Q2: 员工情况很特殊,没有匹配的模板?
**A**: 技能会尝试组合多个模板。你也可以:
```
"这个员工 70% 像防御型,30% 像敏感型,帮我定制一个脚本"
```

### Q3: 能否一次准备多个员工的反馈?
**A**: 可以,但建议分开:
```
会话 1:"帮我准备张工的预反馈(技术强但协作差)"
会话 2:"帮我准备李经理的预反馈(销售业绩不达标)"
```
这样每个员工的脚本会更精准。

### Q4: 生成的脚本太长,记不住?
**A**: 重点看这两个文件:
1. `quick-reference.md` - 浓缩版,1 页纸
2. `feedback-script-[姓名].md` 的【核心反馈】部分

其他部分作为参考,不需要背下来。

### Q5: 对话中忘记了话术怎么办?
**A**: 正常!对话不是背台词。关键是:
- 记住 3 个优点
- 记住 3 个核心问题(用 SBI)
- 记住行动计划

其他话术可以现场灵活发挥。
