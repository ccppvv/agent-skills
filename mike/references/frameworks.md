# Integrated Analysis Frameworks

Comprehensive frameworks combining multiple mental models for deep analysis.

## Five-Dimensional Analysis Framework

A systematic approach to analyzing complex problems from multiple perspectives.

### Dimension 1: Strategic View

**Models**: First Principles, Second-Order Thinking, Inversion, Opportunity Cost

**Focus**: Core purpose and long-term implications

**Questions**:
1. What's the fundamental problem we're solving?
2. What happens after we implement this? (2nd and 3rd order effects)
3. What would guarantee failure, and how do we avoid it?
4. What are we giving up by choosing this path?

**Output**: Strategic framing and key trade-offs

---

### Dimension 2: Systems View

**Models**: Systems Thinking, Feedback Loops, Leverage Points, Stocks & Flows

**Focus**: Interconnections and dynamics

**Questions**:
1. What are the key components and their relationships?
2. What feedback loops are operating (reinforcing/balancing)?
3. Where are the high-leverage intervention points?
4. What stocks are being accumulated/depleted and at what rates?

**Output**: System map and leverage point identification

---

### Dimension 3: Cognitive View

**Models**: Confirmation Bias, Anchoring, Framing, Dunning-Kruger

**Focus**: Potential biases and blind spots

**Questions**:
1. What biases might be affecting my thinking?
2. Am I seeking confirming or disconfirming evidence?
3. How is this framed, and what's the alternative framing?
4. What do I not know that I don't know?

**Output**: Bias checklist and perspective alternatives

---

### Dimension 4: Probabilistic View

**Models**: Bayesian Thinking, Expected Value, Black Swan, Base Rates

**Focus**: Uncertainty and risk

**Questions**:
1. What's the prior probability? How does evidence update it?
2. What's the expected value of each option?
3. What tail risks could occur? How fragile are we?
4. What does the base rate data say?

**Output**: Probability-weighted scenarios and risk assessment

---

### Dimension 5: Economic View

**Models**: Incentives, Supply/Demand, Comparative Advantage, Network Effects

**Focus**: Incentives and resource allocation

**Questions**:
1. What incentives are operating? What behavior do they encourage?
2. What's the supply and demand dynamic?
3. Where do we have comparative advantage?
4. What economic forces will shape outcomes?

**Output**: Incentive analysis and economic viability assessment

---

## Decision-Making Workflow

### Phase 1: Problem Definition (Strategic)
1. Use First Principles to identify core problem
2. Frame from multiple angles (Framing Effect)
3. Define success criteria clearly

### Phase 2: Analysis (Systems + Cognitive + Probabilistic)
1. Map the system (Systems Thinking)
2. Check for biases (Cognitive models)
3. Quantify probabilities (Bayesian Thinking)
4. Calculate expected values

### Phase 3: Generate Options (Strategic + Economic)
1. Brainstorm alternatives (Inversion helpful)
2. Evaluate trade-offs (Opportunity Cost)
3. Check incentive alignment
4. Identify leverage points

### Phase 4: Decision (Probabilistic + Economic)
1. Compare expected values
2. Assess risk tolerance
3. Check reversibility
4. Consider optionality

### Phase 5: Implementation (Systems + Economic)
1. Identify feedback loops to monitor
2. Set up incentives correctly
3. Find leverage points for intervention
4. Plan for 2nd-order effects

### Phase 6: Review (Cognitive + Probabilistic)
1. Update beliefs with outcomes (Bayesian)
2. Check for hindsight bias
3. Extract lessons learned
4. Iterate framework

## Musk's Five-Step Algorithm (五步工作法)

> A strict sequential execution algorithm for building things right. From Elon Musk's practice.
> Full details: [musk-methodology.md](musk-methodology.md)

### The Algorithm (order is sacred)

```
Step 1: Question Requirements (质疑需求)
  ↓
Step 2: Delete Parts & Processes (删减)
  ↓
Step 3: Simplify (简化)
  ↓
Step 4: Accelerate (加速)
  ↓
Step 5: Automate (自动化)
```

### Step 1: Make Requirements Less Dumb

- Assume ALL requirements are stupid, regardless of who提出
- Smart people's requirements are most dangerous — you're less likely to question them
- Ask: Who提出? Why? What happens if we remove it?

### Step 2: Delete Parts and Processes

- Delete aggressively
- Target: < 10% of deleted items need to be restored
- Anti-"just in case" principle

### Step 3: Simplify and Optimize

- **MUST follow Step 2** — never optimize something that shouldn't exist
- "The most common mistake of smart engineers is optimizing a thing that should not exist"

### Step 4: Accelerate

- Only after direction is right (Step 1), simplified (Steps 2-3)
- Tesla Model 3: designed for 5000/week, achieved 7000/week through simplification

### Step 5: Automate

- **MUST be last** — only after all previous steps complete
- Musk's confession: "I've done this in reverse — automate first, then accelerate, then simplify, then realize the requirement was stupid. Total waste."

### Integration with Decision Workflow

Use the Five-Step Algorithm in **Phase 5: Implementation** of the decision workflow:
1. Question the requirements behind your strategic choice
2. Delete unnecessary complexity
3. Simplify what remains
4. Accelerate execution
5. Automate only when stable

### When to Use

- Product design and development
- Process optimization
- System architecture
- Any "building" phase after strategic direction is set

### Key Insight

> "The best part is no part. The best process is no process."

## VDS 快速分析框架

> 来源：道长是名思维贩子《如何思考》系列 | 轻量版问题分析

**定位**：五维框架的前置/轻量版，30分钟完成方向判断。

### 5步问题澄清（输→入→限→困→身）

1. **明确输出**：想要什么？（必须可衡量）
2. **明确输入**：用什么方法？（形成输入-输出逻辑）
3. **加条件**：什么限制？（时间/空间/资源）
4. **加困境**：什么阻碍？（从理想落到现实）
5. **加角色**：你是谁？（具象化问题主体）

### VDS 三刀

- **V（根变量）**：不可替代的2-3个核心因子
- **D（差异）**：能反转为优势的不同点
- **S（策略）**：一句话行动指令（可执行+可证伪）

### 与五维框架的关系

| | VDS | 五维框架 |
|--|-----|---------|
| 用时 | 30分钟 | 1-2小时 |
| 输出 | 1句话策略 | 多维度报告 |
| 关系 | 前置/方向判断 | 深度验证 |

**推荐组合**：VDS 先切方向 → 五维后验深度

## Common Problem Patterns

### "Should I do X?"
1. **Strategic**: First Principles → What problem does X solve?
2. **Probabilistic**: Expected Value → What are likely outcomes?
3. **Cognitive**: Biases → What am I not seeing?
4. **Economic**: Opportunity Cost → What am I giving up?
5. **Systems**: Second-Order → What happens next?

### "Why is X happening?"
1. **Systems**: Feedback Loops → What loops are operating?
2. **Economic**: Incentives → What behavior is incentivized?
3. **Cognitive**: Framing → How am I interpreting this?
4. **Strategic**: First Principles → What's really happening?

### "How do I improve X?"
1. **Systems**: Leverage Points → Where's high impact?
2. **Strategic**: Inversion → What causes failure?
3. **Economic**: Marginal Thinking → What's next improvement?
4. **Probabilistic**: Base Rates → What works typically?

### "What will happen if X?"
1. **Strategic**: Second-Order Thinking → Then what?
2. **Systems**: Feedback Loops → What gets amplified?
3. **Probabilistic**: Base Rates + Bayesian → What's likely?
4. **Cognitive**: Biases → Am I over/under confident?

## Integration Principles

1. **Start Broad, Then Deep**: Use multiple models for overview, then dive deep on most relevant

2. **Look for Convergence**: If multiple models point same direction, confidence increases

3. **Value Disagreement**: When models conflict, explore why - often reveals key insights

4. **Iterate**: Don't expect perfect analysis first pass - refine as you learn

5. **Stay Practical**: Perfect analysis is the enemy of good decision-making

6. **Document**: Write down your reasoning for future learning

---

## Common Pitfalls

1. **Model Overload**: Using too many models creates confusion, not clarity
   - *Solution*: Start with 2-3 most relevant

2. **Confirmation Seeking**: Using models to confirm existing beliefs
   - *Solution*: Actively seek disconfirming models

3. **Inappropriate Application**: Forcing models where they don't fit
   - *Solution*: Check model assumptions against situation

4. **Analysis Paralysis**: Endless analysis prevents action
   - *Solution*: Set decision deadlines, accept uncertainty

5. **Ignoring Implementation**: Perfect analysis, poor execution
   - *Solution*: Include implementation in framework
