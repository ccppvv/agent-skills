# 互联网岗位AI工作流赋能

## 概述

本技能为互联网行业核心岗位提供系统化的AI工作流赋能方案。通过科学的分析框架、丰富的岗位知识库和可直接使用的Prompt模板，帮助个人和团队快速识别AI应用机会，设计定制化工作流，并输出专业的赋能报告。

**核心价值：**
- 🎯 **精准诊断**：基于RICE评分法和三维痛点矩阵，系统识别高价值AI应用场景
- 🛠️ **即用工具**：提供8大岗位的Prompt模板库和AI工具选型矩阵
- 📊 **量化评估**：ROI计算模型和分阶段实施Roadmap，确保落地效果
- 📄 **专业输出**：生成结构化赋能报告，适用于个人提效和企业咨询

**适用场景：**
- 个人希望用AI提升岗位工作效率
- 企业进行岗位级AI赋能时需要系统性方案
- 需要输出专业的AI工作流咨询报告
- 构建岗位级AI应用的最佳实践库

### 1. 岗位工作流分析

基于**任务分层法**，将岗位工作分解为四个层次：

```

执行层 (40-50%)  → 任务执行、内容产出  ← AI高潜力区
```

## SKILL创建引导机制

### 🔍 识别可封装场景

在与用户讨论具体提效场景时，**主动识别**是否可以封装成独立的SKILL。满足以下条件时，应引导用户创建SKILL：

**封装判断标准：**

1. **高复用性** - 场景会被重复使用（频次 ≥ 1次/周）
2. **流程固定** - 有明确的步骤和输入输出
3. **独立完整** - 可以作为独立功能模块
4. **通用价值** - 不仅对当前用户有用，对其他人也有价值

**典型可封装场景示例：**

| 场景类型 | 示例 | 封装价值 |
|---------|------|---------|
| **岗位专用工作流** | "产品经理PRD生成器"、"运营活动策划助手" | 高频使用，流程标准化 |
| **跨岗位通用流程** | "会议纪要生成器"、"邮件润色助手" | 适用多个岗位，复用性强 |
| **行业特定场景** | "电商数据分析"、"SaaS客户成功管理" | 行业知识沉淀，专业性强 |
| **工具链编排** | "内容创作全流程"（选题→撰写→配图→发布） | 多步骤自动化，效率提升显著 |

### 🎯 引导创建流程

当识别到可封装场景时，按以下流程引导用户：

**Step 1: 场景价值确认**
```
"我注意到您提到的[场景名称]是一个高频使用的场景。
这个场景具有很好的封装价值：
- 使用频次：[频次]
- 流程标准化程度：[高/中]
- 预期提效：[百分比]

如果封装成独立的SKILL，您可以：
✅ 一键调用，无需每次重复描述需求
✅ 持续优化Prompt模板，积累最佳实践
✅ 分享给团队成员，实现知识复用

是否需要我帮您创建这个SKILL？"
```

**Step 2: 调用 skill-creator**

用户确认后，立即使用 `Skill` tool 调用 `skill-creator`：

```
Skill tool 参数：
- skill: "skill-creator"
- args: "[场景名称] [场景描述和核心功能]"
```

**示例：**
```
场景：产品经理需要快速生成PRD文档

调用：
Skill(
  skill="skill-creator",
  args="prd-generator 为产品经理提供快速PRD文档生成能力。输入需求要点，自动生成结构化PRD，包含需求概述、用户故事、功能清单、交互流程、验收标准等完整章节。"
)
```

**Step 3: 协助完善SKILL**

创建SKILL后，协助用户完善：
- 补充references文档（Prompt模板、最佳实践）
- 添加示例和使用说明
- 测试和优化输出质量

### ⚠️ 不建议封装的场景

以下场景**不建议**封装成SKILL：

❌ **一次性任务** - 使用频次 < 1次/月
❌ **高度定制化** - 每次需求差异很大，难以标准化
❌ **简单查询** - 可以用一句话Prompt解决的简单任务
❌ **已有现成工具** - 市面上已有成熟的专用工具

**示例：**
```
用户："我需要分析一下这份财报"
→ 不建议封装（一次性任务）

用户："帮我优化这段代码"
→ 不建议封装（已有Copilot等工具）

用户："每天早上给我发送今日待办"
→ 建议封装（高频+流程固定）
```

## 最佳实践

### 1. 分析前的准备

**收集信息：**
- 岗位基本信息（职责、团队规模）
- 典型工作日的任务清单
- 当前痛点和期望改善的方向
- 预算范围和时间要求

**明确目标：**
- 个人提效 vs 团队赋能
- 快速见效 vs 系统优化
- 报告输出 vs 方案实施

### 2. 方案设计原则

**优先级排序：**
1. 高频 + 高痛点 + 高AI适配度的任务
2. 快赢场景（2-4周见效）优先于长期项目
3. 单点工具优先于复杂工作流编排

**工具选择：**
- 从免费/低成本工具开始
- 优先选择通用工具（Claude/GPT）
- 根据ROI逐步升级到专用工具

**实施策略：**
- 小范围试点验证（2-3人，2-4周）
- 快速迭代优化Prompt模板
- 建立内部最佳实践库

### 3. 常见问题处理

**Q: 用户不确定自己的岗位痛点？**
- 引导用户回顾最近1-2周的工作日志
- 使用"三维痛点矩阵"（时间、质量、能力）提问
- 提供岗位知识库中的典型痛点作为参考

**Q: 用户担心AI工具成本？**
- 优先推荐免费工具和试用版本
- 计算ROI，展示投资回报
- 建议从单个高频场景开始，验证效果后再扩展

**Q: 用户不知道如何开始？**
- 推荐"快赢场景"（2-4周见效，提效>50%）
- 提供完整的Prompt模板，降低使用门槛
- 建议从日常高频任务开始（如邮件、文档）

**Q: 团队成员抵触AI工具？**
- 强调"增强而非替代"的定位
- 展示具体的时间节省案例
- 组织培训和分享会，建立信心
