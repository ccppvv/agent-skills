# /batch-type-annotate - Batch Type Annotation

> **Workflow Description**: Add type annotations to all Python files in specified directory

**Detected Patterns**: batch

## Workflow Graph

```
→ DiscoverFiles [batch_1]
→ BatchProcess [batch_2]
  Items: [file1.py, file2.py, ..., fileN.py]
  Operation: AddTypeAnnotations
  Parallel: 4 concurrent files
→ ValidateChanges [batch_3]
```

## Behavioral Flow

1. **Discover Python Files**
   - Scan src/ directory recursively
   - Find all *.py files
   - Exclude __pycache__ and test files

2. **Batch Process** (parallel execution)
   - Process up to 4 files concurrently
   - For each file:
     - Parse AST and identify functions
     - Infer or add type hints for parameters and returns
     - Update file with annotations
   - Track progress across batch

3. **Validate Changes**
   - Run mypy type checker on all modified files
   - Ensure no syntax errors introduced
   - Report summary of changes

## MCP Integration

- **serena MCP**: Code parsing, AST manipulation, and symbol operations

## Usage Examples

```bash
/batch-type-annotate

# Processes all Python files in src/
# Example: 50 files → ~12 parallel batches → completes in 3-5 minutes
```

## Boundaries

**Will:**
- Add type annotations to function parameters and returns
- Process multiple files in parallel for efficiency
- Validate changes with mypy

**Will Not:**
- Modify files outside src/ directory without confirmation
- Add complex generic types without confidence
- Proceed if mypy validation fails
